# 模型对象管理

>*模型对象：定义出全新的统计模型概念，将分析模型抽象化，形成一个个模型对象，模型对象可以被继承和扩展，模型对象的继承和扩展无限制，数据分析师可定义出符合自己需要的分析模型。*

- **模型对象**：模型对象是用户统计、分析、报表维度的起始点；全局统计对象也是用户定制的模板 统计、固定统计的起始点；
- **模板模型**：模板模型是全局模型的子类；用户保存全局模型的分析维度保存而成；
- **选股模型**：固定模型是全局模型的子类；用户保存全局模型的分析维度、分析条件保存而成；
- **分析因子开放性、扩展性**：在架构层面根本解决了开放性的问题，凡是模型定义的分析维度，均可进行筛选、过滤、分析，极大提高了数据分析的各种可能；
- **分析因子钻取深度和广度**：用户可进行任意维度、任何指标层面的钻取分析，从根本上解决数据不同数据分析结果校验的问题；

>*要建立模型对象的重用能力：模型对象可以被继承和扩展，模型子类完全继承父类的所有特性，并可以赋予自己特色；模型对象的继承和扩展没有任何限制，用户可以任意定义符合自己需要的分析模型。*

## 模型对象的维护

1.  **修改全局模型**：管理员可对统计名称、统计时间、单位权限、所属分组以及对象说明进行修改完善；<br>
2.  **删除全局模型**：将删除继承于选定的全局模型对象衍生的子模板/固定模型对象，包括主题对象、主题指标、帮助信息、聊天记里、日志信息等；
3.  **添加主题**：如数据字典中新增了新的主题，则可在此点击添加主题，一切以数据字典为准；<br>
4.  **修改主题**：指修改固定代码对应的扩展代码，其他属性在数据字典中维护；
    - **扩展代码新增**，在界面勾选即可；
    - **扩展代码修改**，譬如在【修改扩展代码】界面基本属性中，修改了 固定代码，则可在先取消勾选原有的扩展代码，然后再勾选扩展代码；
5.  **删除主题**：指定的统计时间口径、单位权限主题不能删除；为保证数据的质量，删除主题将一 并用户保存的删除模板、固定模型对象对应的主题；
6.  **同步主题**：将数据字典中字段的最新字段信息同步到该模型对象；
7.  **主题排序**：主题对象和主题指标在统计分析界面、统计结果、报表中显示的顺序；右键点击主 题即可上移下移；

> *凡是统计模型定义的分析维度，均可进行筛选分析，这极大的提高数据分析的应用深度和广度；解放科技人员生产力，几乎不用理解此数据的用途，一切交由精通市场的分析人员去挖掘数据的潜力，科技人员只需关注提供数据。*

## 选股模型的维护

1. 对当前用户保存的模板模型进行维护；管理员界面不可新增固定模型对象；
2. 模板模型原则上源自于全局统计，但也可由模板、固定模型衍生；
3. 模板模型对象只能修改和删除，目的是让管理员能够控制用户的统计行为，权限等；
4. 模板模型对象用户也可自行维护，在管理个人资源中维护管理；

> *选股模型的用处：市场研究员通过基本面、高频数据、历史数据的分析，保存为固定条件，即可生成对固定，可盘中、盘后生成股票池或策略的校验。*
